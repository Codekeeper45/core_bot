# EXTENDING.md — как собрать своего бота на этом ядре

Ядро даёт готовую инфраструктуру (3 канала, AI-агент, память, медиа, передача менеджеру).
Чтобы получить бота под свою задачу, обычно нужно поменять три вещи: **персону**, **инструменты**
и **подключить каналы**. Ниже — пошагово, в том числе как делать это одним промптом в Claude Code.

---

## TL;DR через Claude Code

Открой проект в [Claude Code](https://claude.com/claude-code) и опиши задачу:

> «Сделай из этого ядра бота — администратора стоматологии. Он отвечает на вопросы об услугах
> и ценах, умеет записать пациента на приём и передаёт сложные случаи менеджеру.»

Claude Code прочитает `AGENTS.md`, перепишет персону в `system_prompt.txt` и создаст нужные
инструменты в `src/tools/`. Для добавления конкретного инструмента есть команда **`/add-tool`**.

> Реалистичное ожидание: персона меняется буквально одним промптом. Инструменты, которые ходят
> в твои внешние системы (CRM, календарь, БД), потребуют уточнений — какой API, какие поля, какая
> авторизация. Это диалог на несколько сообщений, а не одна строка. Зато вся обвязка уже готова.

---

## Шаг 1. Персона бота

Открой `src/agent/prompts/system_prompt.txt` и опиши, кто такой бот и что он делает.
Сохрани полезные правила ядра: ответы plain-text (без Markdown), кратко (2–4 предложения),
и когда звать менеджера.

Пример:
```
Вы — AI-администратор стоматологической клиники «Улыбка».
Помогаете записаться на приём, отвечаете на вопросы об услугах и ценах.
Если пациент жалуется или вопрос требует врача — передавайте менеджеру.
Пишите простым текстом, без разметки, кратко.
```

## Шаг 2. Инструменты под задачу

Инструмент — это действие, которое бот может выполнить (записать на приём, проверить заказ,
создать тикет). Один инструмент = один файл в `src/tools/`. Скопируй `src/tools/exampleEcho.js`
и переделай:

```js
'use strict';

const definition = {
  type: 'function',
  function: {
    name: 'book_appointment',
    description: 'Записать пациента на приём. Зови, когда пациент назвал услугу и удобное время.',
    parameters: {
      type: 'object',
      properties: {
        service: { type: 'string', description: 'Услуга' },
        datetime: { type: 'string', description: 'Дата и время' },
      },
      required: ['service', 'datetime'],
    },
  },
};

async function handler(args, context) {
  // context = { channel, chatId, phone, clientName }
  // здесь — вызов твоего API записи / запись в БД
  return { success: true, message: `Записал на ${args.service}, ${args.datetime}` };
}

module.exports = { definition, handler };
```

Сохрани файл — и всё. Ядро подхватит инструмент автоматически, ничего больше править не нужно.
Проверка:
```bash
node -e "console.log(require('./src/tools').tools.map(t => t.function.name))"
```

Передача диалога живому человеку (`notify_manager`) уже встроена — отдельно делать не надо.

## Шаг 3. Каналы и запуск

1. `cp .env.example .env` и заполни: `OPENROUTER_API_KEY`, `MYSQL_*`, токен(ы) каналов.
2. Подключи каналы — подробно в [README.md](README.md#подключение-каналов):
   - **Telegram** — токен от @BotFather, готов сразу.
   - **WhatsApp** — привязка через страницу `/pair` (ссылку можно отправить кому угодно) или QR.
   - **Instagram** — через Wazzup24.
3. Запусти: `npm start` (или `docker compose up` — поднимет бота вместе с MySQL).

## Чек-лист готовности

- [ ] Переписан `system_prompt.txt` под свою роль.
- [ ] Добавлены нужные инструменты в `src/tools/` (по одному файлу).
- [ ] Удалён пример `src/tools/exampleEcho.js` (если не нужен).
- [ ] Заполнен `.env`, подключён хотя бы один канал.
- [ ] `node --test` зелёный, `npm start` поднимается, `/health` отвечает.
